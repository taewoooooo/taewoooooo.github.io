浏览器工作原理

在浏览器中输入服务器ip地址，就能获取到服务器对应网页资源的index.html文件，浏览器会解析这个html文件，遇到link标签引用了css文件，就从服务器下载对应css文件；遇到script标签就从服务器下载对应js文件，并由浏览器内核将这些资源转化成用户看得见的网页

浏览器内核

浏览器渲染过程

  • DOM树是在解析html文件后生成，同时上方紫色的js代码也会对DOM树进行操作(document.xxx)
  • DOM树和样式规则会共同生成RenderTree，在这里面进行页面展示相关操作
  • 浏览器为了缩短白屏时间，会一边解析一边将已经解析完的部分dom和cssom渲染到页面上
  • 但是js代码由谁来执行呢？ 由JavaScript引擎执行

JavaScript引擎

浏览器内核和JS引擎的关系

浏览器内核就是渲染进程，包含GUI渲染线程和JS引擎线程

  • 渲染引擎决定了浏览器如何显示网页的内容和页面的格式信息
  • JS引擎决定了如何将js代码进行解释编译执行

V8引擎原理

  • 对js源代码进行词法分析和语法分析，生成AST抽象语法树 https://astexplorer.net/
    • 词法分析就是对js代码中的每个独立的词生成对应的对象存于tokens，每个对象记录词的类型和值，比如const生成的就是{type: 'VariableDeclaration', value:'const'}
    • 语法分析就是对上面生成的每个词的对象进行整合，生成AST
  • AST会转化成字节码，而不是01的机器码，因为不同的cpu有不同的处理方式，统一转化为字节码方便接下来的不同编译操作
  • 在Ignition中，会收集一些出现频率高的代码片段，将其直接编译为机器码，提高执行性能
    • 但是如果本次执行不同于之前，则会进行反优化，将机器码转化为字节码，再进行具体操作
    function foo(a, b) {
	return a+b
}
foo(1, 1)
foo(2, 2)
foo('a', 'b') //这次执行不同于之前的加法，数字相加的机器码不能做到字符拼接，所以要反优化回字节码

全局函数执行过程

自己理解的JS代码执行过程

  1. 执行代码前先创建GO(Global Object)对象，将js代码中所有定义的全局变量(var)和函数声明变量放进去（在解析成AST过程中加入）
    1. 此时全局变量的值为undefined
    2. 遇到了函数声明在堆空间创建函数对象，并在GO对象中创建函数名称的属性，指向函数对象的地址
      1. 在函数对象中生成代码块和词法解析确定的父级作用域
  2. 执行js代码
    1. 创建GEC(Globe execution context，全局执行上下文)，压入ECS(Execution context stack, 执行上下文栈)中
      1. GEC中包含VO(Variable Object)，VO指向GO内存地址，依次执行代码，全部执行完毕将GEC弹出栈，js代码执行完毕
    2. 遇到了函数执行(foo()这样的)时
      1. 创建FEC(Function execution context，函数执行上下文)，压入ESC中
        1. FEC中包含VO，指向AO（AO包含函数声明中定义的变量(undefined)、函数声明(地址)等），AO是在创建函数的AST后、函数执行之前创建的
        2. 作用域链，包含本层VO和父级VO，查找变量时一层层往上找，父级是按照函数定义声明的位置决定的，而不是函数执行位置
    3. 依次执行函数体内容，执行完毕将FEC弹出栈

高阶函数

  • 数组中的filter、map、forEach、find、findIndex、reduce等方法
    • filter是按照规则过滤函数，如果结果为true就保留，false就跳过，生成新数组
      • 例如过滤出数组中的偶数： var num1 = num.filter((item, index, num) ⇒ return item%2===0)
    • map是映射，将数组中的所有元素按照某种计算规则映射成新的数组
      • 例如所有元素×10： var num1 = num.map((item) ⇒ return item*10)
    • forEach是遍历数组元素
    • find是找到第一个能让内置函数返回true的元素值
      • 找到第一个值为11的元素： var num_11 = num.find((item) ⇒ return item===11)
    • findIndex返回的是第一个找到的元素下标
    • reduce是累加，preValue是上一次函数调用返回的值，item是正在遍历的数组元素，initValue是初始化preValue的值
      • 累加所有元素： var total = num.reduce((preValue, item) ⇒ {return preValue+item}, initValue)

// call函数传入的参数： thisBings是调用call的函数里面的this需要绑定的对象，第二个是参数列表
Function.prototype.taewooCall = function(thisArg, ...args) {
  
  // 判断thisBings是否是数字、字符串、布尔值、null、undefined等不是对象的内容
  //    如果是数字字符串布尔值，则用Object()将其转化为对应的包装类，比如Number、String、Boolean等
  //    如果是null或undefined，则thisBings指向的对象应该为window
  //    如果thisBings是0的话，也会走window这个判断，所以应该改为thisBings!==undefined && thisBings!==null
  thisArg = (thisArg!==undefined && thisArg!==null) ? Object(thisArg) : window
  
  // 获取谁调用了这个taewooCall，并将其存为thisBings的fn属性，用于下面调用
  // 是谁调用的taewooCall，谁就是taewooCall的this，本例中foo调用，故this指向foo，
  // 我们需要让foo函数中的this指向thisArg(bar)，就需要将foo作为bar的属性，并用bar.foo()的执行方式隐式绑定foo中的this为bar
  thisArg.fn = this

  // 用bar.foo()的方式执行foo，此时foo中的this已经指向了bar
  var result = thisArg.fn(...args)

  // 用完了fn就把fn这个属性从thisBings(bar)中删掉
  delete thisArg.fn

  return result
}

function bar() {

bind使用拓展运算符

Function.prototype.taewooBind = function(thisArg, ...bindArgs) {

  thisArg= (thisArg!== undefined && thisArg!== null) ? Object(thisArg) : window
  var fn = this
  thisArg.fn = fn

  // 闭包
  function proxyFn(...newArgs) {
    var finalArgs = [...bindArgs, ...newArgs]
    var result = thisArg.fn(...finalArgs)
    return result
  }

  return proxyFn
}

Function.prototype.taewooBind = function(thisArgs) {
  thisArgs = (thisArgs !== null || thisArgs !== undefined) ? Object(thisArgs) : window;
  let self = this;
  // 获取arguments的除了第一个参数剩下的参数，因为第一个参数是指定this，后面才是需要的参数
  // 而arguments是类数组，无法使用数组原型上的slice方法，所以用call借用方法使用（改变slice中的this）
  let args = Array.prototype.slice.call(arguments, 1);
  return function() {
    // 返回的函数中的参数都留下
    let newArgs = Array.prototype.slice.call(arguments);
    let finalArgs = args.concat(newArgs);
    console.log(finalArgs);
    let result = self.apply(thisArgs, finalArgs);

    return result;
  }
}

bind考虑new的覆盖this问题

Function.prototype.taewooBind = function(context) {
  context = (context !== null || context !== undefined) ? Object(context) : window;
  let self = this;
  let args = Array.prototype.slice.call(arguments, 1);
  
  let fBound = function() {
    let newArgs = [].slice.call(arguments);
    let finalArgs = args.concat(newArgs);
    // 当作为构造函数时，忽略bind绑定的context，this指向new出来的实例
    return self.apply(this instanceof fBound ? this : context, finalArgs);
  }
  // 实现继承
  fBound.prototype = Object.create(self.prototype);
  Object.defineProperty(fBound.prototype, 'constructor', {
    enumerable: true,
    configurable: true,
    writable: true,
    value: fBound
  });
  return fBound;
}

Array.prototype.taewooSlice = function(start, end) {
  // slice默认转换的是函数内部this指向的类数组或数组
  var arr = this
  start = start || 0
  end = end || arr.length
  
  var newArr = []
  for(var i=start; i<end; i++){
    newArr.push(arr[i])
  }

  return newArr
}

构造函数的缺点

  • 每创建一个新的Person对象实例，都要为里面的eating和running变量创建一片新的函数对象的空间
  • 即p1.eating ≠ p2.eating 浪费大量空间和性能
  • 解决方案是用原型

原型

  • prototype的constructor属性的描述符中的enumerable为false，意味着通过foo.prototype无法直接遍历显示constructor属性
  • constructor属性指向的是函数本身，即foo.prototype.constructor = foo 类似一个循环
  • Function.prototype = p1.__proto__ = p2.__proto__
  • new创建出来的对象内部的属性[[prototype]]就是__proto__
  • __proto__ 属性其实是Object.getPrototypeOf 的getter/setter

function inheritPrototype(subType, superType) {
  // ES6语法
  subType.prototype = Object.create(superType.prototype);
  Object.defineProperty(subType.prototype, 'constructor', {
    enumerable: false,
    configurable: true,
    writable: true,
    value: subType
  });
}
