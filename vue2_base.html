如何使用Vue

方式一：CDN引入

  • CDN是内容分发网络
  • 是指通过互相连接的网络系统，利用最靠近每个用户的服务器，更快更可靠的将数据传送给用户，提供高性能低成本的网络内容
  • <script src="https://cdn.jsdelivr.net/npm/vue@2.6.14/dist/vue.js"></script>

方式二：下载和引用

npm install vue -g

声明式和命名式

  • 命名式编程关注的是“how to do”，声明式编程关注的是“what to do”，由框架完成how的过程
  • 原生js和jQuery使用一条条代码来给浏览器命令 → 命令式
  • Vue是给传入的对象中声明需要的内容，模板template，数据data，方法methods → 声明式

MVVM模型

Model-View-ViewModel

  • Model → javaScript操作，模型，数据对象data
  • View → DOM，模板页面
  • ViewModel → Vue

计算属性computed

复杂data的处理方式：

  1. 对多个数据进行处理结合，再通过插值语法显示，可以在模板中使用表达式 → 缺点是维护困难，大量重复代码，没有缓存
  2. 调用methods方法，没有缓存
  3. 使用计算属性computed，是Vue传入的options中的一个属性
<body>
  <div id="app">
    <h2>{{firstName+lastName}}</h2>
    <h2>{{fullNameMethods()}}</h2>
    <h2>{{fullNameComputed}}</h2>
  </div>

  <script src="./js/vue.js"></script>
  <script>
    const app = new Vue({
      el: "#app",
      data: {
        firstName: "Kwon",
        lastName: "taewoo",
      },
      methods: {
        fullNameMethods(){
          return this.firstName + '' + this.lastName;
        }
      },
      computed: {
        fullNameComputed(){
          return `${this.firstName}${this.lastName}`;
        }
      },
    });
  </script>
</body>

计算属性无法直接进行传参

如果有传参数的需求比如说做数据筛选功能可以使用闭包函数（也叫匿名函数）实现，例如

computed: {
  myfilter() {
    return function(index){
      return index的操作;         
    }           
  } 
}

computed和methods的对比

  • computed是有缓存的，多次使用计算属性时，只会执行一次对应的computed中方法，当依赖数据发生改变后，需要重新计算一次 methods没有缓存，每次执行都要重新执行对应methods中方法
  • computed在{{}}不需要使用() method在{{}}中需要使用()

计算属性的getter和setter

一般v-model搭配计算属性需要setter和getter，否则会报错

computed的完整写法：

computed: {
  fullName:{
    get: function(){
      return xxx;
    },
    set: function(newValue){
      let names = newValue.split(' ');
      this.firstName = names[0];
      this.lastName = names[1];
    }
  }
}

this.fullName; //调用getter
this.fullName = "abc def"; //调用setter

<input type="text" v-model="fullName">

数组/对象中的响应式方法

  1. arr.push("a");在数组最后添加一个或多个元素
  2. arr.pop(); 删除数组的最后一个元素
  3. arr.shift(); 删除数组的首个元素
  4. arr.unshift("a"); 在数组首位前添加一个或多个元素
  5. splice方法
    • arr.splice(1,2);从下标为1的元素开始(包含这个元素)删除两个元素，第二个元素为空则删除到最后
    • arr.splice(1, 2, "a", "b"); 从下标为1的元素开始（包含其），将后面两个元素替换为"a"和"b"
    • arr.splice(1, 0, "a"); 将下标为1的元素替换为"a"
  6. arr.sort(); 排序 降序
  7. arr.reverse(); 逆置
  8. Vue.set(要修改的对象，索引值，修改后的值)

数组中的非响应式操作

  1. arr[2] = 1; 通过索引值更改元素 用splice替换

runtime-compiler/only的区别

Vue程序运行过程

template →解析→ ast(抽象语法树) →编译→ render(functions) → virtual dom → UI(真实dom)

runtime-compiler (main.js代码)

import Vue from "vue"
import App from "./App"

new Vue({
	el: "#app",
	// template->ast->render->vdom->ui
	template: "<App/>",
	components: {App}
})

runtime-only（main.js代码）

import Vue from "vue"
import App from "./App"

new Vue({
	el: "#app",
	//省去了template->ast的过程，直接从render->vdom->ui,性能更好，代码量更少
	render: h => h(App)
})

其中h是形参函数，真实的函数时createElement()

例如 createElement("h2", {class: 'box'}, ["hello world", createElement(...)]);

标签h2里的内容会替换掉el挂载的app元素(<div class="app"></div>)

方式二：createElement(组件名)

例如 createElement(App)

导航守卫

导航守卫 | Vue Router

正如其名， vue-router 提供的导航守卫主要用来通过跳转或取消的方式守卫导航。有多种机会植入路由导航过程中：全局的, 单个路由独享的, 或者组件级的。 记住 参数或查询的改变并不会触发进入/离开的导航守卫。你可以通过来应对这些变化，或使用 beforeRouteUpdate 的组件内守卫。 你可以使用 router.beforeEach 注册一个全局前置守卫： 当一个导航触发时，全局前置守卫按照创建顺序调用。守卫是异步解析执行，此时导航在所有守卫 resolve 完之前一直处于 等待中 。 每个守卫方法接收三个参数： 确保 next 函数在任何给定的导航守卫中都被严格调用一次。它可以出现多于一次，但是只能在所有的逻辑路径都不重叠的情况下，否则钩子永远都不会被解析或报错。这里有一个在用户未能验证身份时重定向到 /login 的示例： 在 2.5.0+ 你可以用 router.beforeResolve 注册一个全局守卫。这和 router.beforeEach 类似，区别是在导航被确认之前， 同时在所有组件内守卫和异步路由组件被解析之后 ，解析守卫就被调用。 你也可以注册全局后置钩子，然而和守卫不同的是，这些钩子不会接受 next 函数也不会改变导航本身： 你可以在路由配置上直接定义 beforeEnter 守卫： 这些守卫与全局前置守卫的方法参数是一样的。 最后，你可以在路由组件内直接定义以下路由导航守卫： beforeRouteEnter 守卫 不能 访问 this ，因为守卫在导航确认前被调用，因此即将登场的新组件还没被创建。 不过，你可以通过传一个回调给 next 来访问组件实例。在导航被确认的时候执行回调，并且把组件实例作为回调方法的参数。 注意 beforeRouteEnter 是支持给 next 传递回调的唯一守卫。对于 beforeRouteUpdate 和 beforeRouteLeave 来说， this 已经可用了，所以 不支持 传递回调，因为没有必要了。 这个离开守卫通常用来禁止用户在还未保存修改前突然离开。该导航可以通过 next(false) 来取消。 导航被触发。 在失活的组件里调用 beforeRouteLeave 守卫。 调用全局的 beforeEach 守卫。 在重用的组件里调用 beforeRouteUpdate 守卫 (2.2+)。 在路由配置里调用 beforeEnter。 解析异步路由组件。 在被激活的组件里调用 beforeRouteEnter。 调用全局的 beforeResolve 守卫 (2.5+)。 导航被确认。 调用全局的 afterEach 钩子。 触发 DOM 更新。 调用 beforeRouteEnter 守卫中传给 next 的回调函数，创建好的组件实例会作为回调函数的参数传入。

全局导航守卫

需求： 想要在多个路由组件间切换时可以随着更新对应的网页标题 方法一：直接在每个组件的vue文件中添加created()函数，在组件一开始创建时就更改标题内容，缺点是麻烦，如果有100个组件就要在每个组件中都写created函数

<script>
export default {
  name: "About",
  created(){
    document.title = '关于';
  }
}
</script>

方法二：全局导航守卫，在index.js中更改

  • 前置守卫是beforeEach(to, from, next)，在跳转之前执行的内容
  • 后置守卫是afterEach(to, from)，跳转之后执行的内容，不需要主动调用next函数
  • 两者都是全局守卫
const routes = [
{
  path: '/About',
  component: About,
  // meta是元数据，描述数据的数据
  meta: {
    title: '关于'
  },
}];



// 前置守卫
router.beforeEach((to, from, next) => {
  // 从from跳转到to
  // matched[0]是防止嵌套获取不到父路由的meta
  document.title = to.matched[0].meta.title;

  // 必须运行next，跳转到下一状态
  next();
});

mutations提交风格

  1. commit提交单个变量，即this.$store.commit('decrementCount', 10); 将10传入mutations中的increment(state, count)的count中
  2. commit提交包含type属性的对象，在increment(state, payload)中使用payload.count获取传入count的值5

mutations响应规则

  • 所有在state初始定义的属性都会加入到vue的响应式系统中，监听这些属性的变化，一旦变化就会通知界面中所有用到该属性的地方更新数据
  • 如果后加入一些属性：state.info['address']='aaa'，info中后加入的address属性就不会被加入到响应式系统中
  • 解决方法→使用Vue.set方法： Vue.set(state.info, 'address', 'aaa')
  • 删除state中对象的属性：delete方法做不到响应式，应该使用Vue.delete(对象名，要删除的属性名) 是响应式的

actions

  • mutations如果处理异步操作则不会再dev-tool中捕获到改变
  • 需要将异步操作放在actions中执行才能被工具捕获到数据变化
  • dispatch提交的是actions中的方法，commit提交的是mutations中的方法
  • dispatch和commit都可以传入第二个参数
  • 可以结合promise coderwhy P138后半部分
// index.js
mutations: {
  updateInfo(state){
    state.info.name = 'aaaaaa';
  }
},
// 处理异步操作
actions: {
  // context 等价于state
  actionsUpdateInfo(context){
    setTimeout(() => {
      // 这里提交的是mutations中的方法
      context.commit('updateInfo');

    }, 1000)
  }
}

// App.vue
methods: {
    updateInfo() {
      // 异步操作在App.vue中不使用commit，在actions中使用commit
      // 这里dispatch的是actions中的方法
      this.$store.dispatch('actionsUpdateInfo'); 
    }
  }

export function request(config) {

  const instance = new axios.create({

    timeout:5000
  });

  // 2. axios拦截器
  // 2.1 请求拦截的作用
  instance.interceptors.request.use(config => {
    //拦截后需要将拦截下来的请求数据返回发送
    return config;
  }, err => {

  })

  // 响应拦截器

    // 拦截后需要将拦截下来处理成的结果返回
    return res.data
  }, err => {
    console.log(err)
  })

  //  3. 发送真正的网络请求，是一个promise对象
  return instance(config)
}

上传方法一：

  1. vue create 项目名称
  2. npm run serve / npm run dev

    如果没有config文件夹，或者看package.json文件中的"scripts"对象里面有'serve'，则是vue-cli3.x，不能用npm run dev 需要使用npm run serve

  3. 在github上创建一个仓库，赋值仓库地址， 然后在控制台输入 git clone 仓库地址，会创建一个和仓库中项目名一样的文件夹
  4. 将之前vue create创建的项目里除了.git和node_modules文件之外的所有都复制到新生成的文件夹中
  5. cd进入新文件夹，输入git add . ，git commit -m '初始化项目' ，此时提交到本地
  6. git push 提交到git服务器
  7. 以后每次修改代码后都重复第五步和第六步提交到服务器

上传方法二：(不需要复制)

  1. vue create 项目名称
  2. git remote add origin 仓库地址
  3. git push -u origin master
